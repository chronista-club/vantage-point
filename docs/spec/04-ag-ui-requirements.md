# AG-UI プロトコル基盤 要件定義

> WebUIとClaude Codeを繋ぐ架け橋

## 背景

- MCPは Agent↔Tools の接続を標準化
- AG-UIは Agent↔User の接続を標準化
- 現状のPermissionRequest等はMCP経由のポーリングで回りくどい
- AG-UI採用によりリアルタイムなUI更新が可能に

---

## 要件一覧

### ライフサイクル管理

#### REQ-AGUI-001: 実行開始イベント

**概要**: エージェント実行の開始をWebUIに通知する

**背景**: ユーザーがエージェントの動作状態を把握するため

**受け入れ条件**:
- [ ] RunStartedイベントをWebSocketで送信できる
- [ ] WebUIでエージェント実行中の表示が出る
- [ ] 実行IDが一意に識別できる

**関連設計**: design/02-ag-ui-architecture.md

---

#### REQ-AGUI-002: 実行終了イベント

**概要**: エージェント実行の終了（成功/エラー）をWebUIに通知する

**背景**: ユーザーが処理完了を把握し、結果を確認できるようにする

**受け入れ条件**:
- [ ] RunFinishedイベントで正常終了を通知できる
- [ ] RunErrorイベントでエラー終了を通知できる
- [ ] WebUIで実行結果のステータスが表示される

**関連設計**: design/02-ag-ui-architecture.md

---

#### REQ-AGUI-003: ステップ進捗イベント

**概要**: 実行中のステップ単位の進捗をWebUIに通知する

**背景**: 長時間実行時にユーザーが現在の処理段階を把握できるようにする

**受け入れ条件**:
- [ ] StepStartedイベントでステップ開始を通知
- [ ] StepFinishedイベントでステップ完了を通知
- [ ] WebUIで進捗バーまたはステップ表示が更新される

**関連設計**: design/02-ag-ui-architecture.md

---

### メッセージストリーミング

#### REQ-AGUI-010: テキストメッセージストリーミング

**概要**: エージェントからのテキストメッセージをリアルタイムでストリーミング表示する

**背景**: 応答生成中もユーザーがリアルタイムで内容を確認できるようにする

**受け入れ条件**:
- [ ] TextMessageStart/Content/Endイベントを送信できる
- [ ] WebUIでトークン単位のストリーミング表示ができる
- [ ] キャンセル操作で生成を中断できる

**関連設計**: design/02-ag-ui-architecture.md

---

### ツールコール

#### REQ-AGUI-020: ツールコール通知

**概要**: エージェントがツールを呼び出す際にWebUIに通知する

**背景**: ユーザーがエージェントの行動を透過的に確認できるようにする

**受け入れ条件**:
- [ ] ToolCallStart/End イベントを送信できる
- [ ] ツール名と引数がWebUIに表示される
- [ ] ツール実行結果がWebUIに表示される

**関連設計**: design/02-ag-ui-architecture.md

---

#### REQ-AGUI-021: ツール承認（Human-in-the-Loop）

**概要**: 危険なツール実行前にユーザーの承認を求める

**背景**: ファイル変更やシステム操作等、確認が必要な操作を安全に実行する

**受け入れ条件**:
- [ ] PermissionRequestイベントで承認UIを表示できる
- [ ] ユーザーが承認/拒否を選択できる
- [ ] 選択結果がエージェントに返却される
- [ ] タイムアウト時のデフォルト動作が設定可能

**関連設計**: design/02-ag-ui-architecture.md

---

### 状態同期

#### REQ-AGUI-030: 状態スナップショット

**概要**: エージェントの現在状態をWebUIと同期する

**背景**: WebUI再接続時やページリロード時に状態を復元する

**受け入れ条件**:
- [ ] StateSnapshotイベントで完全な状態を送信できる
- [ ] WebUI接続時に現在状態が復元される
- [ ] 複数クライアント間で状態が一貫する

**関連設計**: design/02-ag-ui-architecture.md

---

#### REQ-AGUI-031: 状態デルタ更新

**概要**: 状態変更を差分で効率的に同期する

**背景**: 大きな状態オブジェクトを毎回送信せず、変更部分のみ送信する

**受け入れ条件**:
- [ ] StateDeltaイベントで差分更新を送信できる
- [ ] WebUIで差分がマージされ状態が更新される
- [ ] 差分適用の順序が保証される

**関連設計**: design/02-ag-ui-architecture.md

---

### トランスポート

#### REQ-AGUI-040: WebSocketトランスポート

**概要**: AG-UIイベントをWebSocket経由で配信する

**背景**: 既存のvp daemonのWebSocket基盤を活用する

**受け入れ条件**:
- [ ] 既存のWebSocket接続でAG-UIイベントを送受信できる
- [ ] イベントはJSON形式でシリアライズされる
- [ ] 接続断時の再接続でイベント欠損がない

**関連設計**: design/02-ag-ui-architecture.md

---

#### REQ-AGUI-041: イベント型定義（Rust）

**概要**: AG-UIイベントをRust構造体として定義する

**背景**: 型安全なイベント処理を実現する

**受け入れ条件**:
- [ ] 全イベントタイプがRust enumで定義される
- [ ] serde でJSON シリアライズ/デシリアライズ可能
- [ ] TypeScript型定義も生成可能（将来）

**関連設計**: design/02-ag-ui-architecture.md
