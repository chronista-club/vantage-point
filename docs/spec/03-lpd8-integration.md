# LPD8 MIDI コントローラー連携仕様

## 概要

Akai LPD8/LPD8 mk2を使ったVantage Pointの物理操作インターフェース。
8つのパッドで、プロジェクト切り替えとセッション操作を行う。

## パッドマッピング

```
┌─────────────────────────────────────┐
│  LPD8 (Program 1 = VP Mode)         │
├─────────┬─────────┬─────────┬───────┤
│  PAD 5  │  PAD 6  │  PAD 7  │ PAD 8 │
│  (40)   │  (41)   │  (42)   │ (43)  │
│ Cancel  │  Reset  │   -     │   -   │
│  [赤]   │  [橙]   │  [off]  │ [off] │
├─────────┼─────────┼─────────┼───────┤
│  PAD 1  │  PAD 2  │  PAD 3  │ PAD 4 │
│  (36)   │  (37)   │  (38)   │ (39)  │
│ Proj 1  │ Proj 2  │ Proj 3  │ Proj 4│

└─────────┴─────────┴─────────┴───────┘
```

### プロジェクト切り替え（PAD 1-4）

| パッド | Note | アクション | ポート | LED色 |
|--------|------|------------|--------|-------|
| PAD 1 | 36 | Project 1 に切り替え | 33000 | 緑 |
| PAD 2 | 37 | Project 2 に切り替え | 33001 | 緑 |
| PAD 3 | 38 | Project 3 に切り替え | 33002 | 緑 |
| PAD 4 | 39 | Project 4 に切り替え | 33003 | 緑 |

### セッション操作（PAD 5-8）

| パッド | Note | アクション | LED色 |
|--------|------|------------|-------|
| PAD 5 | 40 | チャットキャンセル | 赤 |
| PAD 6 | 41 | セッションリセット | 橙 |
| PAD 7 | 42 | 未割当 | off |
| PAD 8 | 43 | 未割当 | off |

## プログラム管理

LPD8には4つのProgram（バンク）があり、ハードウェアで切り替え可能。

| Program | 用途 |
|---------|------|
| Program 1 | VP用（`vp lpd8 write`で設定） |
| Program 2 | VPサブ（予備） |
| Program 3-4 | ユーザー任意（DAW等） |

## コマンド

### LPD8設定

```bash
# MIDI出力ポート一覧
vp lpd8 ports

# VP設定をLPD8 Program 1に書き込み
vp lpd8 write

# Program 2に書き込み
vp lpd8 write -p 2

# アクティブプログラムを切り替え
vp lpd8 switch 1
```

### MIDI監視

```bash
# MIDI入力ポート一覧
vp midi-ports

# MIDI監視開始（インタラクティブ）
vp midi

# 特定のデーモンに接続してMIDI監視
vp midi -P 33000
```

## SysExプロトコル

VP設定の書き込みにはMIDI SysExを使用。

| 項目 | 値 |
|------|-----|
| Manufacturer ID | 0x47 (Akai) |
| Model ID | 0x7F 0x75 (LPD8) |

### コマンド

| コマンド | バイト | 説明 |
|----------|--------|------|
| Send Program | 0x61 | プログラムデータを送信 |
| Set Active | 0x62 | アクティブプログラムを変更 |
| Get Program | 0x63 | プログラムデータを取得 |
| Get Active | 0x64 | アクティブプログラム番号を取得 |

## 動作仕様

### プロジェクト切り替え

1. PADを押すと対応するポートのデーモンにWebSocket接続
2. 既にそのプロジェクトが起動していない場合は何もしない（将来的には起動も検討）
3. 接続先が変わると、UIは自動的に新しいプロジェクトのセッションを表示

### セッション操作

- **Cancel**: 現在のAI応答生成を中断
- **Reset**: 新しいセッションを開始（履歴クリア）
