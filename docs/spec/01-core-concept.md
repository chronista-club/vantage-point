# Vantage Point - コアコンセプト

> 開発行為を拡張する

## ビジョン

Vantageは、開発者とAIの協働体験を拡張するプラットフォーム。
場所やデバイスに縛られず、シームレスに開発を継続できる環境を提供する。

## コア体験

### 1. AIと対話する
没入空間でClaudeと協働。単なるチャットではなく、開発の「相棒」として。

### 2. 思考を空間に配置・整理
頭の中のアイデアを可視化し、空間に配置して整理する。

### 3. 複数の情報を並べる
コード、ドキュメント、ターミナルを自由に配置。
必要な情報を必要な時に、必要な場所で。

### 4. シームレスにつながる
上記すべてが断絶なく連携する。

## 対話スタイル

### AI主導の選択肢提示

**基本パターン**: AIが選択肢を提示 → ユーザーが選ぶ

- 「次どうしますか？ A / B / C」
- タップで選択、必要な時だけテキスト入力
- 移動中でも軽い入力で対話継続

**テキスト入力は補助的**
- 選択肢にない操作
- 詳細な指示
- 自由な発想の入力

### 協調モード（3段階）

| モード | 説明 | ユースケース |
|--------|------|-------------|
| **協調** | ユーザーと一緒に進める | 設計・意思決定が必要な作業 |
| **委任** | 任せて、途中経過・結果を確認 | 定型作業、リファクタリング |
| **自律** | 完全に任せる | バックグラウンドタスク |

## シームレス要件

### 課題
「出かけなきゃいけない時に、続きが打てない」

### 解決
- Mac → iPad/iPhone で作業継続
- デバイス間で状態が同期
- 移動中は軽い入力（選択肢タップ）で対話継続

## 対応プラットフォーム

| プラットフォーム | 位置づけ |
|-----------------|---------|
| **Mac** | メイン開発環境 |
| **iPad/iPhone** | 移動中の継続 |
| **Vision Pro** | 未来の開発環境（最重要） |

### 設計方針

**共通化より各プラットフォーム最適化**

- 各デバイスの特性を活かしたUI
- Vision Proの空間体験を最大限に
- 無理な共通化で体験を犠牲にしない

## 開発優先順位

1. **AI主導の選択肢UI** - Macでスタイル確立
2. **空間に情報を並べる体験** - Vision Pro向け

## 技術方針

- **CLI-First**: Rust製CLIをコアとして開発
- **Claude CLI統合**: Agent Serviceとして組み込み
- **段階的拡張**: Mac CLI → WebView → Vision Pro
