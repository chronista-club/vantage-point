# Vantage Point - ユーザージャーニー

> 一日の開発体験をシームレスに

## 理想の一日

```
起床 → MIDIパッド → 作業開始 → Vision Pro → 机 → 移動中 → カフェ → 帰宅
     ↓
  すべて一つのワークスペース上で継続
```

### シーン1: 起床・作業開始

1. 椅子に座る
2. AKAI LPD8（MIDIコントローラー）のPadを叩く
3. UIが開く
4. 作業開始

**ポイント**: ワンアクションで即座に作業モードへ

### シーン2: Vision Proでソファ作業

- 机からソファへ移動
- Vision Proを被る
- **同じワークスペース**がそのまま空間に展開
- 没入環境で集中作業

### シーン3: 机に戻る

- Vision Proを外す
- Macの前に戻る
- **そのまま継続** - 状態は完全同期

### シーン4: 移動中（コンビニへ）

- iPhoneを取り出す
- AIの作業経過を確認
- 選択肢をタップして進める
- または見守るだけ

**ポイント**: 移動中は軽い操作で継続

### シーン5: カフェでiPad作業

- iPadを開く
- **同じワークスペース**が開く
- しっかり作業を継続

### シーン6: 帰宅後

- 再びMacの前へ
- **そのまま作業継続**
- 一日通して一つのワークスペース

## デバイス遷移マトリクス

| From / To | Mac | Vision Pro | iPad | iPhone |
|-----------|:---:|:----------:|:----:|:------:|
| **Mac** | - | シームレス | シームレス | シームレス |
| **Vision Pro** | シームレス | - | シームレス | シームレス |
| **iPad** | シームレス | シームレス | - | シームレス |
| **iPhone** | シームレス | シームレス | シームレス | - |

**すべての遷移がシームレス** - これがVantageの核心

## 入力トリガー

### 物理トリガー: AKAI LPD8

MIDIコントローラーを開発のコマンドセンターとして活用。

| Pad | アクション |
|-----|-----------|
| Pad 1 | UI起動 / ワークスペース1 |
| Pad 2 | ワークスペース2 |
| Pad 3 | ワークスペース3 |
| ... | カスタマイズ可能 |

**ユースケース**:
- 朝: Pad叩いて即作業開始
- 作業中: Pad切り替えでプロジェクト間を瞬時に移動
- 将来: その他のハードウェア連携も検討

### 各デバイスでの入力

| デバイス | 主な入力 | 補助入力 |
|---------|---------|---------|
| Mac | キーボード、MIDI | マウス、音声 |
| Vision Pro | 視線+ピンチ、音声 | 仮想キーボード |
| iPad | タッチ、Apple Pencil | キーボード |
| iPhone | タッチ | 音声 |

## 状態同期の要件

### 同期対象
- 現在のワークスペース状態
- AIとの会話コンテキスト
- 開いているファイル・ドキュメント
- 配置（空間レイアウト）
- 協調モード設定

### 同期タイミング
- **リアルタイム**: 可能な限り即時同期
- **オフライン対応**: 一時的な切断でも継続可能

## 技術的含意

このユーザージャーニーから導かれる技術要件:

1. **中央サーバー必須** - 状態の一元管理
2. **リアルタイム通信** - WebSocket等
3. **各プラットフォームネイティブクライアント** - 最適な体験のため
4. **オフライン対応** - 一時的な切断への耐性
5. **MIDI対応** - macOSでのハードウェア連携
